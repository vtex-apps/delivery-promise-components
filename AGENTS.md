# AGENTS.md — delivery-promise-components

## Repository Purpose

`vtex.delivery-promise-components` is a VTEX IO Store Framework app that exports
blocks enabling shoppers to set a postal code, choose a shipping method (delivery
vs. pickup), and select a pickup point. It is gated to stores using the Delivery
Promise platform service (closed beta). The main exported blocks are
`shopper-location-setter`, `shipping-method-selector`, `pickup-point-selector`,
and `availability-badges`.

## Sources of Truth

Read these files before editing — they define the app contract:

| File | What it defines |
|---|---|
| `manifest.json` | App identity (`vtex.delivery-promise-components@1.1.1`), declared builders (`react`, `messages`, `docs`, `store`), platform dependencies, and policies |
| `store/interfaces.json` | Store Framework block interfaces, component bindings, and content schemas (props) |
| `react/*.tsx` | Exported React components — one file per block entrypoint |
| `react/DeliveryPromiseContext.tsx` | Shared React context: all state, actions, and side-effect logic lives here |
| `react/typings/` | VTEX IO platform typings; do not modify without Toolbelt regeneration |
| `react/package.json` | React builder dependency boundary; do not mix with root `package.json` |
| `messages/en.json` | Canonical i18n strings (reference locale for `intl-equalizer`) |
| `docs/README.md` | User-facing block reference: props, CSS handles, integration examples |
| `README.md` | Contributor guide: prerequisites, how to run/test/publish |

## Verified Commands

These commands exist in the repo (verified from `Makefile`, `package.json`):

```sh
make dev          # yarn install (root + react) + vtex setup
make lint         # yarn lint (eslint, no --fix)
make test         # cd react && yarn test (vtex-test-tools)
make coverage     # cd react && yarn test --coverage
make check        # lint + test (pre-PR gate)
make link         # vtex link (active workspace — confirm with vtex whoami first)
make run          # alias for make link
make clean        # remove node_modules + coverage
```

Root scripts (via `yarn`):
- `yarn lint` — ESLint across `.ts/.tsx/.js/.jsx`
- `yarn format` — Prettier (writes); use `make format-check` for CI-safe check
- `yarn lint:locales` — intl-equalizer (validates message completeness)
- `yarn test` — delegates to `cd react && yarn test`

## Autonomy Limits

- **Never** run `vtex link`, `vtex publish`, `vtex deploy`, workspace promotion, or
  any account/workspace-changing Toolbelt command without explicit confirmation.
- **Never** modify `react/typings/` files directly — they are regenerated by
  `vtex setup --typings --ignore-linked`.
- **Never** modify `manifest.json` `version` — version bumps go through
  `vtex release <patch|minor|major> stable`.
- **Ask before** running `vtex setup --tooling` — it can overwrite `react/.eslintrc`
  and root Prettier/ESLint config if custom rules are present.
- **Ask before** adding or removing declared `builders` in `manifest.json` — each
  builder change affects what BuilderHub validates and what the platform packages.
- Secrets and credentials live outside this repo (VTEX account/workspace context).
  Do not commit tokens, account names, or workspace names.
- The `docs/` folder is owned by `@vtex-apps/technical-writers`; the `messages/`
  folder is owned by `@vtex-apps/localization`. Coordinate changes accordingly.
